# rk3588-driver-monitor

> 作品名称：基于RK3588的车载AI安全监测与语音控制系统

---

## 项目简介

传统驾驶员监测系统（DMS）普遍存在"检测即报警、无法持续干预、缺乏行为理解"的痛点，难以应对连续低头、双手离盘等复合风险场景。本作品面向智能座舱安全需求，基于 RK3588 边缘计算平台，设计并实现了一套"**端侧实时感知 — 风险动态累积 — PC 端认知分析 — 分级座舱干预**"的智能驾驶认知决策闭环系统。

技术架构上，端侧融合 YOLOv8-Pose 姿态估计与 RetinaFace 人脸检测，结合车辆运动状态与环境传感，实现低头、离盘、打电话、嘴部风险等多行为实时监测；PC 端部署 DeepSeek 大模型，对风险事件序列进行时序认知分析，生成可解释干预策略与驾驶员画像；态势感知与决策看板实现感知、决策、执行、反馈的全链路可视化闭环。

核心创新体现在三方面：
1. **认知风险累积机制**：通过滑动窗口融合多行为时序特征，实现 L1/L2/L3 三级风险动态升级，解决传统 DMS 单帧误报与重复报警问题；
2. **端云混合决策架构**：端侧负责 40ms 级实时预警与座舱级干预，PC 端承担非实时策略解释与画像更新，二者解耦协同且端侧始终掌握安全兜底权；
3. **可解释、可追溯的端-PC 协同验证链路**：支持一键状态重置、多模块同步诊断与长时间稳定运行。

实测端侧 NPU 推理吞吐 25 FPS，视觉检测耗时约 40 ms，PC 端认知分析延迟 4.9~7.8 s。系统经过多轮长时间演示与异常场景验证，具备实时感知、可解释决策和现场稳定演示能力。相比传统规则式 DMS，本系统通过时序认知机制将单帧误报转化为分级预警，显著提升了干预有效性与驾驶员接受度。

## 系统部署

本系统采用端-PC 协同架构，**两端需同时运行**。

### 环境要求

- **端侧**：RK3588 开发板（Linux），Python 3.8+，RKNN Toolkit
- **PC 端**：Python 3.8+，Flask，DeepSeek API Key

### 1. 克隆仓库

```bash
git clone https://github.com/JH3234501237-UI/rk3588-driver-monitor.git
cd rk3588-driver-monitor
```

### 2. 配置 DeepSeek API Key

编辑 `pc/start_edge_agent.bat`，将 `YOUR_DEEPSEEK_API_KEY` 替换为有效 Key。

### 3. 启动 PC 端

进入 `pc` 目录，运行后端服务：

```bash
cd pc
python edge_agent.py
```

### 4. 启动端侧（一键清场 + 保护期）

在 RK3588 端侧执行演示脚本：

```bash
cd edge
./start_demo.sh
该脚本会自动完成：清理旧事件/锁文件 → 重置风险状态 → 拉起风险看门狗与状态上传 → 设置 8~15 秒启动保护期。
```
